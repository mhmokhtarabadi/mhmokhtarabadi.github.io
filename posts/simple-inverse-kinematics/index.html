A simple two DoF inverse kinematics in MATLAB.

1. Introduction

As you may already know, there are two major ways to solve Inverse Kinematics (IK) problems. Analytical, and Numerical approaches. In these two approaches, the solver will find a set of coordinates which specify the robot configuration in order to satisfy some constraints, for example position of a point or orientation of a link of the robot.

However, in this blog we want to how a two DoF robot can travel a path to in order for it’s end point to reach a desired position. We then write a MATLAB program that shows the real-time animation of this robot. We can specify the desired position by clicking on a point in robot feasible regin, and robot will try to reach that point.

2. Mathematical Equations

From robot arm shown in figure bellow, the position of it’s end point (point A) is:

\begin{equation} \begin{split} &x_A = L_1 \cos(\theta_1) + L_2 \cos(\theta_2) \\ &y_A = L_1 \sin(\theta_1) + L_2 \sin(\theta_2) \end{split} \end{equation}

So if the desired position is $(x_D, y_D)$, then the vector from point A to point D would be:

\begin{equation} \vec{d} = \begin{bmatrix} x_D-x_A \\ y_D-y_A \end{bmatrix} \end{equation}

Now if we assume that the velocity of point A to be $v_A$, if this point want to reach point D, the $x$ and $y$ component of it’s velocity vector should be:

\begin{equation} \begin{split} &{v_A}_x = \frac{v_Ad_x}{\sqrt{d_x^2+d_y^2}} \\ &{v_A}_y = \frac{v_Ad_y}{\sqrt{d_x^2+d_y^2}} \end{split} \end{equation}

Which in equation above, $d_x$ and $d_y$ are $x$ and $y$ components of vector $\vec{d}$. However, to control the robot configuration we need to know the angular velocity of it’s DoFs ($\dot{\theta}_1$, $\dot{\theta}_2$). So if we take a time derivative from point A’s position (equation #1), we have:

\begin{equation} \begin{split} \dot{x}_a = \frac{\delta x_a}{\delta \theta_1}\dot{\theta}_1 + \frac{\delta x_a}{\delta \theta_2}\dot{\theta}_2 \\ \dot{y}_a = \frac{\delta y_a}{\delta \theta_1}\dot{\theta}_1 + \frac{\delta y_a}{\delta \theta_2}\dot{\theta}_2 \end{split} \longrightarrow \begin{bmatrix} \dot{x}_a \\ \dot{y}_a \end{bmatrix} = \begin{bmatrix} \frac{\delta x_a}{\delta \theta_1} & \frac{\delta x_a}{\delta \theta_2} \\ \frac{\delta y_a}{\delta \theta_1} & \frac{\delta y_a}{\delta \theta_2} \end{bmatrix} \begin{bmatrix} \dot{\theta}_a \\ \dot{\theta}_a \end{bmatrix} \end{equation}

Which matrix $\begin{bmatrix}\frac{\delta x_a}{\delta \theta_1} & \frac{\delta x_a}{\delta \theta_2} \\ \frac{\delta y_a}{\delta \theta_1} & \frac{\delta y_a}{\delta \theta_2}\end{bmatrix}$ is called Jacobian matrix ($J$), and in this robot is:

\begin{equation} J = \begin{bmatrix} \frac{\delta x_a}{\delta \theta_1} & \frac{\delta x_a}{\delta \theta_2} \\ \frac{\delta y_a}{\delta \theta_1} & \frac{\delta y_a}{\delta \theta_2} \end{bmatrix} = \begin{bmatrix} -L_1 \sin (\theta_1) & -L_2 \sin (\theta_2) \\ L_1 \cos (\theta_1) & L_2 \cos (\theta_2) \end{bmatrix} \end{equation}

Now, from equation 4, 5, we can calculate $\dot{\theta}_1$ and $\dot{\theta}_1$:

\begin{equation} \begin{bmatrix} \dot{\theta}_1 \\ \dot{\theta}_2 \end{bmatrix} = \begin{bmatrix} -L_1 \sin (\theta_1) & -L_2 \sin (\theta_2) \\ L_1 \cos (\theta_1) & L_2 \cos (\theta_2) \end{bmatrix}^{-1} \begin{bmatrix} \dot{x}_a \\ \dot{y}_a \end{bmatrix} \end{equation}

Which $\begin{bmatrix}\dot{x}_a \\ \dot{y}_a\end{bmatrix}$ is $\begin{bmatrix}{v_A}_x \\ {v_A}_y\end{bmatrix}$ in equation 3.

That’s it. Now if we know the current and desired position of point A, we can change $\theta_1$ and $\theta_2$ such that these two points match. Because we want to implement it discretely, $\theta_1$ and $\theta_2$ will be updated as bellow:

\begin{equation} \begin{split} \theta_1^{(i)} = \theta_1^{(i-1)} + \dot{\theta}_1^{(i)} T_s \\ \theta_2^{(i)} = \theta_2^{(i-1)} + \dot{\theta}_1^{(i)} T_s \end{split} \end{equation}

Which $T_S$ is out program sample time (0.02 s), and $i$ is iteration number.

clc; clear; close;

% Given Data
L1 = 1.0;   % m
L2 = 0.5;   % m

xD = -0.2;  % initial desired position (x) - m
yD = 0.8;   % initial desired position (y) - m

theta1 = deg2rad(50);   % initial theta1 - rad
theta2 = deg2rad(-40);  % initial theta2 - rad

velocity = 4;   % end point velocity - m/s

sample_time = 0.02;   % s

c1 = 0.3;   % theta filtering constant

min_det = 0.1;  % minimum value of the determinant of the Jacobian matrix

We a first-order low-pass filter with $C_1$ constant to prevent sudden change in $\theta_1$ and $\theta_2$ values:

\begin{equation} \theta_j^{(i)} = C_1 \theta_j^{(i)} + (1-C_1) \theta_j^{(i-1)} \end{equation}

When the end point of our robot (A) will get close to the edges of feasible reigns, the determinant of Jacobian matrix will get close to zero, and makes it inaccurate to calculate it’s inverse in equation 6. When the Jacobian matrix determinant is less than min_det, we will use Jacobian matrix from previous iteration.

% Main Loop

f1 = figure;

J = [1 0; 0 1]; % initial Jacobian matrix
dtheta(1, :) = [0; 0];  % initial dtheta - rad/s^2
i = 2;
while true
    % equation (1)
    xA = L1*cos(theta1) + L2*cos(theta2);
    yA = L1*sin(theta1) + L2*sin(theta2);

    % saving end points for plotting path
    x(i-1) = xA;
    y(i-1) = yA;

    % equation (2)
    dx = xD - xA;
    dy = yD - yA;

    % eduation (3)
    dxA = velocity*dx / sqrt(dx^2 + dy^2);
    dyA = velocity*dy / sqrt(dx^2 + dy^2);

    % equation (5)
    J_pre = J;
    J = [-L1*sin(theta1) -L2*sin(theta2); ...
          L1*cos(theta1) L2*cos(theta2)];
    if abs(det(J)) < min_det
        J = J_pre;
    end

    % equation (6)
    dtheta(i, :) =  J \ [dxA; dyA];
    % equation (8)
    dtheta(i, :) = c1*dtheta(i, :) + (1 - c1)*dtheta(i-1, :);

    %equation (7)
    theta1 = wrapTo2Pi(theta1 + dtheta(i, 1)*sample_time);
    theta2 = wrapTo2Pi(theta2 + dtheta(i, 2)*sample_time);
    i = i + 1;

    hold on

    line(x2, y2, 'linewidth', 2)
    plot(x, y, 'color', 'k', 'LineStyle','-.', 'LineWidth', 1)
    scatter(xD, yD, 20, 'k', 'filled', 'o')

    line([0 0.25], [0 0], 'color', 'k', 'LineStyle', '--', 'LineWidth', 0.5)
    text(0.2*cos(theta1/2), 0.2*sin(theta1/2), '\theta_1')
    line([0 0.25]+L1*cos(theta1), [0 0]+L1*sin(theta1), 'color', 'k', 'LineStyle', '--', 'LineWidth', 0.5)
    text(L1*cos(theta1)+0.2*cos(theta2/2), L1*sin(theta1)+0.2*sin(theta2/2), '\theta_2')

    C = get (gca, 'CurrentPoint');  % recieving clicked position
    % saving clicked position as desired values
    xD = C(1, 1);
    yD = C(1, 2);

    pause(sample_time)
end
